## 1.1 Kennzeichen von Variablen

- Wert (Bitmuster, Byteorder)
- Typ (Anzahl der Bytes, Interpretation Bitmuster)
- Addresse des erstes Bytes im Speicher
- Name
- Wenn man eine Variable nicht initialisiert hat sie den Wert der vorher drin stand
  Gilt nur wenn es innerhalb der main-Methode definiert wird, ansonsten automatisch 0
- Wenn man einen Array mit weniger Elementen definiert als in der angegebenen Länge, wird automatisch mit 0 aufgefüllt

## 1.2 Datentyp char

__Datentyp char__

- Speicherung von Zeichen nach ascii-Code
  Bsp: bei `a` enthält dieser dann 0x41 und 61
- findet auch Verwendung als int mit Verarbeitungsbreite von 1 Byte
- wenn man einen char mit einem Zeichen belegen will verwendet man ``
- wenn man einen char mit mehreren Zeichen belegen verwenden wir " "
  jede solche Zeichenkette wird automatisch mit einer Terminierenden 0 beendet

```c
char name[] = "Hans";
```

Es werden für den Array 5 Bytes reserviert, aufgrund der terminierenden 0

## 1.3 Bezeichner

- müssen mit einem Buchstaben oder einem Unterstrich beginnen
- aus Buchstaben, Ziffern und Unterstrich
- case sesitive
- frei wählbar
- sollen gut dokumentieren, besser kurz
- einige reservierte Schlüsselwörter die man nicht verwenden darf

## 1.4 Ausgabe

- Standardfunktion printf mit  ```#include <stdio.h>```
- benötigt als Startpunkt eine main-Funktion
- return 0 markiert das Ende
- \n zur Ausgabe in einer neuen Zeile
- Um Variablen auszugeben wird ein Platzhalter benutzt
  Beginnt mit % gefolgt von der Ausgabenformatierung
- %6d - dezimal, 6 stellig ausgegeben mit führenden Leerzeichen
  %06d - 6 stellig mit führenden Nullen
  %.2f - float mit 2 Nachkommastellen
  %04x - hexadezimal
  (siehe Tabelle Formatsteuerzeichen)

## 1.5 Speicherung

Hinweis: Wenn man in C eine Zahl mit einer führenden 0 defininiert, wird diese als Oktalzahl (zur Basis 8) interpretiert (bei hexadezimal 0x).

Um die Zahl 0x12AB abzuspeichern benötigt man 2 Byte. Auf intel wird es als AB12 (little endian) abgespeichert, auf motorola 12AB (big endian).
